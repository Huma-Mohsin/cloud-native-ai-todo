# cloud-native-ai-todo
Mastering AI-native engineering by building a scalable, Kubernetes-deployed Todo system through iterative specs and Model Context Protocol (MCP).
Cloud-Native AI Todo Evolution 🚀
This repository contains the complete 5-phase journey of building an AI-native Todo system. Starting from a simple Python console app, this project evolves into a fully-featured, cloud-native AI chatbot deployed on Kubernetes.

🏗️ Project Overview
The core philosophy of this project is Spec-Driven Development. Every feature is first defined in a specification file and then implemented using Claude Code and Spec-Kit Plus. No manual coding was performed; the focus remains on system architecture and reusable intelligence.

🛠️ Tech Stack
AI Engine: Claude Code & OpenAI Agents SDK

Backend: Python (FastAPI, SQLModel)

Frontend: Next.js 15+ & Tailwind CSS

Database: Neon Serverless PostgreSQL

Infrastructure: Docker, Kubernetes (Minikube/DOKS), Helm

Integration: MCP (Model Context Protocol), Kafka, Dapr

📅 Road Map (The 5 Phases)
[x] Phase I: In-Memory Python Console App (Current Stage)

[ ] Phase II: Full-Stack Web Application (Next.js + FastAPI)

[ ] Phase III: AI-Powered Chatbot via MCP Server

[ ] Phase IV: Local Kubernetes Deployment (Minikube)

[ ] Phase V: Advanced Cloud-Native Deployment (Kafka & Dapr)
